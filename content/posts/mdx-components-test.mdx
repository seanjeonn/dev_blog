## 소개

이 포스트는 **여러 MDX 컴포넌트를 한 번에 검증**하기 위한 종합 테스트 문서입니다.  
`Callout`, `Tabs`, 테이블, 코드 블록(TS/JS/Bash), 체크리스트, 이미지까지 포함하며, **Next.js 15 App Router + TypeScript** 기준의 간단한 API/페이지 예제도 제공합니다.  
중급 웹 개발자가 실제 블로그 글에 바로 복붙해도 문제없도록 **문서화 친화적 구조**로 구성했습니다.

<Callout type="info" title="이 문서는 테스트용입니다">
실서비스 문서에 넣기 전, 본문 내 모든 컴포넌트가 올바르게 렌더링되는지 확인하세요. 글 상단/하단/중간의 요소들이 서로 간섭하지 않는지 점검합니다.
</Callout>

## 사전지식

- **환경**: Next.js 15(App Router), TypeScript 5+, Node.js 20, Tailwind, shadcn/ui  
- **문법**: MDX + 커스텀 컴포넌트(예: `<Callout/>`, `<Tabs/>`)  
- **목표**: 각 요소의 렌더링/스타일/접근성/코드 하이라이트 정상 동작 확인

## Tabs 컴포넌트 예시

<Tabs defaultValue="typescript">
  <TabsList>
    <TabsTrigger value="typescript">TypeScript</TabsTrigger>
    <TabsTrigger value="javascript">JavaScript</TabsTrigger>
    <TabsTrigger value="python">Python</TabsTrigger>
  </TabsList>

  <TabsContent value="typescript">
    ### TypeScript 타입 안전성

    ```typescript
    interface User {
      id: string;
      name: string;
      email: string;
      role: "admin" | "user";
    }

    const createUser = (data: Omit<User, "id">): User => {
      return {
        id: crypto.randomUUID(),
        ...data,
      };
    };

    const user = createUser({
      name: "John Doe",
      email: "john@example.com",
      role: "user",
    });
    ```

    TypeScript는 컴파일 타임에 타입 체크를 수행하여 런타임 에러를 방지합니다.
  </TabsContent>

  <TabsContent value="javascript">
    ### JavaScript 런타임 검증

    ```javascript
    const createUser = (data) => {
      if (!data.name || !data.email) {
        throw new Error("Name and email are required");
      }

      return {
        id: crypto.randomUUID(),
        name: data.name,
        email: data.email,
        role: data.role || "user",
      };
    };

    const user = createUser({
      name: "John Doe",
      email: "john@example.com",
      role: "user",
    });
    ```

    JavaScript는 런타임에 유효성을 검증합니다.
  </TabsContent>

  <TabsContent value="python">
    ### Python 데이터클래스

    ```python
    from dataclasses import dataclass
    from typing import Literal
    import uuid

    @dataclass
    class User:
        id: str
        name: str
        email: str
        role: Literal["admin", "user"]

    def create_user(
        name: str,
        email: str,
        role: Literal["admin", "user"] = "user"
    ) -> User:
        return User(
            id=str(uuid.uuid4()),
            name=name,
            email=email,
            role=role
        )

    user = create_user(
        name="John Doe",
        email="john@example.com",
        role="user"
    )
    ```

    Python의 데이터클래스는 타입 힌트를 제공합니다.
  </TabsContent>
</Tabs>
